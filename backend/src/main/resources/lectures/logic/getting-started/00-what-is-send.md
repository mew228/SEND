# What SEND Means

SEND stands for **Strategic Economic Node Development**.

It is a system that lets you build economic algorithms using simple building blocks called nodes. Instead of writing code, you will build strategies by placing nodes on a canvas and connecting them together.

Each node does one small job. When you connect multiple nodes, they form a full strategy that the system can run.

:::sublecture { "id":"what-you-are-building", "title":"What You Are Building" }

# What You Are Building

In SEND, you are building **strategies**.

A strategy is just a set of steps:

- Get data (like a stock price)  
- Process or compare that data  
- Make a decision  

For example:

- Get the current price  
- Compare it to a number  
- Decide whether to buy  

You are taking an idea and turning it into something the system can actually execute.

:::sublecture { "id":"how-send-works", "title":"How SEND Works" }

# How SEND Works

You will work on a canvas where you build your strategy visually.

You will:

- Drag and drop nodes onto the canvas  
- Move them to organize your layout  
- Connect them to control how data flows  

When you connect nodes, you are telling the system what happens next.

Different nodes have different roles:

- Some get data  
- Some calculate or transform values  
- Some compare values  
- Some trigger actions  

When combined, they create a full decision system.

:::sublecture { "id":"why-send-exists", "title":"Why SEND Exists" }

# Why SEND Exists

Most tools either hide how strategies work or require coding to build them.

SEND is built so you can:

- See every step of your strategy  
- Build complex logic without coding  
- Learn by actually creating systems  

You are not just learning concepts. You are building real strategies, testing them, and improving them step-by-step.

## What to remember

SEND is about turning ideas into systems.

> Idea → Steps → Strategy

If you can break an idea into steps, you can build it in SEND.
